Para extrair o texto do PDF anexo de forma organizada e utilizá-lo em um script Tampermonkey (JavaScript), evitando a extração truncada e sem lógica, é importante usar uma abordagem que vá além do simples "extrair tudo como texto corrido". O documento que você enviou é estruturado, com tabelas, campos e seções bem delimitadas, o que permite criar uma extração mais inteligente e orientada a regras.

Estratégias recomendadas
1. Use uma biblioteca avançada de extração de PDF em JavaScript

Bibliotecas como Aspose.PDF para JavaScript permitem extrair o texto de PDFs, inclusive em aplicações web e scripts de automação, com maior controle sobre o resultado.

Com ela, você pode extrair o texto de cada página e processar o conteúdo linha a linha, mantendo a separação de tabelas e campos.

Exemplo básico de uso com Aspose.PDF:

javascript
// Supondo que você já tenha carregado a biblioteca Aspose.PDF para JS
var fileReader = new FileReader();
fileReader.onload = function(event) {
    // Extrai o texto do PDF
    var json = AsposePdfExtractText(event.target.result, 'nome_do_arquivo.pdf');
    if (json.errorCode == 0) {
        // Aqui você recebe o texto extraído, pronto para ser processado
        console.log(json.extractText);
    } else {
        console.error(json.errorText);
    }
};
// fileInput é o input[type="file"] do seu script Tampermonkey
fileReader.readAsArrayBuffer(fileInput.files[0]);
O resultado será uma string com o texto extraído, que você pode dividir em linhas e processar conforme a lógica do seu script.

2. Crie regras de extração e processamento

Após extrair o texto, defina regras para identificar e separar as informações (por exemplo, usando expressões regulares para capturar datas, valores, títulos de seções, etc.).

Isso pode ser feito diretamente em JavaScript, processando a string linha a linha e montando objetos ou arrays estruturados.

Exemplo de regra simples para capturar valores das tabelas:

javascript
const linhas = textoExtraido.split('\n');
const resultados = [];
linhas.forEach(linha => {
    // Exemplo: capturar linhas que começam com nome de verba
    if (linha.match(/^AUXILIO ALIMENTAÇÃO|AUXILIO REFEIÇÃO|MULTA DO ARTIGO/)) {
        const partes = linha.split(/\s+/);
        resultados.push({
            verba: partes[0] + ' ' + partes[1],
            valor: partes[partes.length - 1]
        });
    }
});
Você pode refinar essas regras para cada seção do documento, como "Histórico Salarial", "Faltas e Férias", "Demonstrativo de Verbas", etc.

3. Ferramentas online e modelos de extração

Ferramentas como o Calculojuridico.com.br permitem criar modelos de extração baseados em regras, onde você define exatamente quais informações quer buscar e em quais linhas ou padrões elas aparecem.

Essa abordagem pode ser adaptada para seu script: crie funções que busquem padrões específicos no texto extraído.

Resumo das melhores práticas
Evite o simples "extrair tudo": sempre processe o texto extraído com regras que reconheçam padrões, títulos de seções, tabelas e campos.

Utilize bibliotecas especializadas (Aspose.PDF, PDF.js) para garantir que o texto seja extraído com a estrutura de páginas e linhas preservada.

Implemente funções de pós-processamento em JavaScript para transformar o texto em objetos estruturados, prontos para uso no seu Tampermonkey.

Se necessário, crie um modelo de extração com base em exemplos do seu documento, identificando os campos e padrões mais relevantes para o seu caso.

Assim, você terá uma extração organizada, lógica e pronta para automação em seu script, sem os problemas comuns de truncamento e mistura de dados.

Pontos de melhoria identificados
O script utiliza pdf.js para extração do texto, mas a concatenação dos textos das páginas (textContent.items.map(item => item.str).join(' ')) pode gerar perda de estrutura (quebra de colunas, linhas, tabelas).

A segmentação atual por regex é boa, mas pode falhar em documentos com pequenas variações de cabeçalho ou formatação.

A análise dos dados extraídos depende de padrões fixos, o que pode não ser suficiente para todos os tipos de planilhas ou sentenças.

Proposta de otimização para extração mais robusta
1. Preservação de estrutura de linhas
Em vez de juntar todos os textos da página com espaço, monte o texto linha a linha, respeitando a ordem vertical dos itens. Isso pode ser feito agrupando os itens de textContent.items por coordenada Y (posição vertical):

javascript
function agruparPorLinha(items) {
    const linhas = {};
    items.forEach(item => {
        // Arredonda o valor de y para agrupar textos na mesma linha
        const y = Math.round(item.transform[5]);
        if (!linhas[y]) linhas[y] = [];
        linhas[y].push({ x: item.transform[4], str: item.str });
    });
    // Ordena linhas por y (de cima para baixo)
    return Object.values(linhas)
        .sort((a, b) => b[0].y - a[0].y)
        .map(linha => 
            linha.sort((a, b) => a.x - b.x).map(i => i.str).join(' ')
        );
}
No processamento de cada página, substitua o join(' ') por algo como:

javascript
const linhasPagina = agruparPorLinha(textContent.items);
textoCompleto += linhasPagina.join('\n');
Isso preserva as linhas e colunas das tabelas, facilitando a análise posterior.

2. Segmentação contextual aprimorada
Amplie os marcadores de início/fim das seções usando regexes mais tolerantes a pequenas variações.

Use fallback para identificar tabelas pelo padrão de múltiplos números e colunas, caso o cabeçalho varie.

3. Parsing de tabelas e valores
Após a segmentação, use expressões regulares e splitting por múltiplos espaços para identificar tabelas, por exemplo:

javascript
function extrairTabelaResumo(texto) {
    const linhas = texto.split('\n');
    const tabela = [];
    let capturando = false;
    for (const linha of linhas) {
        if (linha.match(/^Descrição.*Bruto Devido/)) {
            capturando = true;
            continue;
        }
        if (capturando && linha.trim() === '') break;
        if (capturando) {
            const partes = linha.trim().split(/\s{2,}/);
            if (partes.length >= 4) {
                tabela.push({
                    descricao: partes[0],
                    bruto: partes[1],
                    juros: partes[2],
                    total: partes[3]
                });
            }
        }
    }
    return tabela;
}
A mesma lógica pode ser aplicada para outras tabelas do PDF, como "Histórico Salarial", "Demonstrativo de Verbas", etc.

4. Validação cruzada
Após extrair os dados, valide se os totais batem com os valores finais apresentados na planilha.

Se possível, implemente alertas para divergências ou ausência de campos obrigatórios.

5. Logs detalhados e exportação para debug
Mantenha e amplie a opção de exportar o texto extraído para facilitar ajustes futuros nos padrões de extração.

Resumo das recomendações
Agrupe o texto por linha durante a extração para preservar a estrutura tabular.

Aprimore as regexes de segmentação para maior tolerância a variações de formatação.

Implemente parsing de tabelas por splitting de múltiplos espaços e validação de colunas.

Valide os dados extraídos cruzando totais e campos obrigatórios.

Facilite o debug mantendo logs e exportação do texto bruto.

Essas melhorias vão tornar a extração do CALC.js mais robusta, confiável e adaptável a diferentes modelos de planilha trabalhista, minimizando erros de captação e facilitando a manutenção do script.
