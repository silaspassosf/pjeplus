Beleza — então fica Python mesmo.

Só dois ajustes importantes no que eu passei (pra funcionar liso no Firefox/Windows):

1) No manifesto do native host (`...NativeMessagingHosts\...json`), o campo **`path` precisa apontar para um executável**, então a forma correta é apontar para o `python.exe` **e chamar o script via argumento** não é suportado nesse JSON. O jeito certo é criar um **.bat** (ou um wrapper `.cmd`) e apontar o `path` para ele (ou então empacotar em exe). O MDN deixa claro que o browser “lança o app nativo” definido em `path` e conversa via stdio. [developer.mozilla](https://developer.mozilla.org/en-US/docs/Mozilla/Add-ons/WebExtensions/Native_messaging)

2) Para evitar dor de cabeça com ID, mantenha o `gecko.id` fixo como eu sugeri.

Abaixo vai um “pacote final” (copiar/colar) já com esses detalhes.

## 1) Extensão (infojud_bridge)

### `manifest.json`
```json
{
  "manifest_version": 3,
  "name": "InfoJud Bridge",
  "version": "0.1.0",
  "description": "Recebe URLs via Native Messaging e abre no Firefox normal.",
  "permissions": ["nativeMessaging", "tabs"],
  "host_permissions": ["<all_urls>"],
  "background": { "scripts": ["background.js"] },
  "browser_specific_settings": {
    "gecko": { "id": "infojud_bridge@local" }
  }
}
```

### `background.js`
```js
const HOST = "br.com.seuapp.infojud_bridge";

let port;
function connect() {
  port = browser.runtime.connectNative(HOST);

  port.onMessage.addListener(async (msg) => {
    try {
      if (!msg || msg.action !== "open_url") {
        port.postMessage({ ok: false, error: "Unknown or missing action" });
        return;
      }

      const url = String(msg.url || "");
      if (!url.startsWith("http://") && !url.startsWith("https://")) {
        port.postMessage({ ok: false, error: "Invalid url" });
        return;
      }

      const tab = await browser.tabs.create({ url, active: true });
      port.postMessage({ ok: true, action: "opened", url, tabId: tab.id });
    } catch (e) {
      port.postMessage({ ok: false, error: String(e) });
    }
  });

  port.onDisconnect.addListener(() => {
    // tenta reconectar se o host reiniciar
    setTimeout(connect, 1000);
  });
}

connect();
```

Carregue em `about:debugging#/runtime/this-firefox` → “Load Temporary Add-on…” → selecione `manifest.json`.

## 2) Native host (Python)

### `native_host.py`
(igual ao anterior; mantém TCP 127.0.0.1:8765 e repassa `open_url` para a extensão)

```python
import sys
import json
import struct
import threading
import socket

HOST = "127.0.0.1"
PORT = 8765

def read_native_message():
    raw_len = sys.stdin.buffer.read(4)
    if len(raw_len) == 0:
        return None
    msg_len = struct.unpack("<I", raw_len)[0]
    data = sys.stdin.buffer.read(msg_len)
    if not data:
        return None
    return json.loads(data.decode("utf-8"))

def send_native_message(obj):
    data = json.dumps(obj).encode("utf-8")
    sys.stdout.buffer.write(struct.pack("<I", len(data)))
    sys.stdout.buffer.write(data)
    sys.stdout.buffer.flush()

def handle_client(conn):
    try:
        buf = b""
        while True:
            chunk = conn.recv(4096)
            if not chunk:
                return
            buf += chunk
            while b"\n" in buf:
                line, buf = buf.split(b"\n", 1)
                line = line.strip()
                if not line:
                    continue
                req = json.loads(line.decode("utf-8"))

                if req.get("action") == "open_url":
                    send_native_message({"action": "open_url", "url": req.get("url", "")})
                    conn.sendall((json.dumps({"ok": True, "status": "sent"}) + "\n").encode("utf-8"))
                else:
                    conn.sendall((json.dumps({"ok": False, "error": "unknown action"}) + "\n").encode("utf-8"))
    except Exception as e:
        try:
            conn.sendall((json.dumps({"ok": False, "error": str(e)}) + "\n").encode("utf-8"))
        except:
            pass
    finally:
        conn.close()

def tcp_server():
    s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
    s.bind((HOST, PORT))
    s.listen(5)
    while True:
        conn, _ = s.accept()
        threading.Thread(target=handle_client, args=(conn,), daemon=True).start()

def main():
    threading.Thread(target=tcp_server, daemon=True).start()

    while True:
        msg = read_native_message()
        if msg is None:
            break
        # log em stderr (stdout é reservado ao Native Messaging)
        sys.stderr.write(f"[native-host] from extension: {msg}\n")
        sys.stderr.flush()

if __name__ == "__main__":
    main()
```

## 3) Wrapper .bat (necessário no Windows)

Crie `run_native_host.bat` (no mesmo diretório do `native_host.py`):

```bat
@echo off
"C:\Caminho\Para\python.exe" "C:\Caminho\Para\native_host.py"
```

Use o **python.exe** que você quer (pode ser do seu venv).  
Você também pode usar `py -3` se preferir, mas apontar direto para o python é mais estável.

## 4) Registrar o host no Firefox (Windows)

Crie a pasta (se não existir):
`%APPDATA%\Mozilla\NativeMessagingHosts\`

Crie o arquivo:
`%APPDATA%\Mozilla\NativeMessagingHosts\br.com.seuapp.infojud_bridge.json`

Conteúdo (ajuste o path do .bat):
```json
{
  "name": "br.com.seuapp.infojud_bridge",
  "description": "InfoJud Bridge Native Host",
  "path": "C:\\Caminho\\Para\\run_native_host.bat",
  "type": "stdio",
  "allowed_extensions": ["infojud_bridge@local"]
}
```

Esse é o formato exigido pelo Native Messaging no Firefox: o browser lança o executável em `path` e troca mensagens via stdio. [developer.mozilla](https://developer.mozilla.org/en-US/docs/Mozilla/Add-ons/WebExtensions/Native_messaging)

## 5) Teste rápido (do seu robô / qualquer Python)

```python
import socket, json

def open_url(url):
    s = socket.create_connection(("127.0.0.1", 8765), timeout=5)
    s.sendall((json.dumps({"action":"open_url", "url": url}) + "\n").encode("utf-8"))
    resp = s.recv(4096).decode("utf-8").strip()
    s.close()
    return resp

print(open_url("https://example.com"))
```

Se estiver tudo ok, o Firefox abre a aba.

***

## Se der erro, os 3 checks mais comuns
1) O host JSON está no caminho certo (`%APPDATA%\Mozilla\NativeMessagingHosts\...`).  
2) O `gecko.id` da extensão bate com `allowed_extensions`.  
3) O `path` do host aponta para um executável válido (no nosso caso, o `.bat`). [developer.mozilla](https://developer.mozilla.org/en-US/docs/Mozilla/Add-ons/WebExtensions/Native_messaging)

Quando isso estiver abrindo URLs, o próximo passo é: a extensão coletar dados da página InfoJud e devolver um JSON (aí você substitui o seu scraping Selenium por scraping dentro do Firefox normal).
