## Objetivo
Criar uma **extensão mínima do Firefox** + um **native host Python** (Native Messaging) para que o seu robô (Selenium/Python) envie uma URL do InfoJud para o **Firefox normal (perfil logado)** abrir, e (por enquanto) receber apenas um ACK (`opened`). Depois a gente evolui para coletar dados da página e devolver JSON.

Isso evita WebDriver no InfoJud (onde você está sendo bloqueado como automatizado), mas mantém o seu fluxo totalmente automatizado via IPC.

Fontes do conceito/formatos: Native Messaging e `runtime.sendNativeMessage/connectNative` no MDN. [developer.mozilla](https://developer.mozilla.org/en-US/docs/Mozilla/Add-ons/WebExtensions/Native_messaging)

***

# Parte A — Extensão Firefox (mínima)

## 1) Estrutura de pastas
Crie uma pasta (ex.: `infojud_bridge/`) com estes arquivos:
- `manifest.json`
- `background.js`

### `manifest.json` (Manifest V3)
```json
{
  "manifest_version": 3,
  "name": "InfoJud Bridge",
  "version": "0.1.0",
  "description": "Recebe URLs via Native Messaging e abre no Firefox normal.",
  "permissions": ["nativeMessaging", "tabs"],
  "host_permissions": ["<all_urls>"],
  "background": {
    "scripts": ["background.js"]
  }
}
```

### `background.js` (abre a URL e responde)
```js
// Conecta ao native host
const PORT_NAME = "br.com.seuapp.infojud_bridge";
let port = browser.runtime.connectNative(PORT_NAME);

port.onMessage.addListener(async (msg) => {
  try {
    if (!msg || !msg.action) {
      port.postMessage({ ok: false, error: "Missing action" });
      return;
    }

    if (msg.action === "open_url") {
      const url = String(msg.url || "");
      if (!url.startsWith("http://") && !url.startsWith("https://")) {
        port.postMessage({ ok: false, error: "Invalid url" });
        return;
      }

      // abre em nova aba (janela normal do Firefox)
      const tab = await browser.tabs.create({ url, active: true });
      port.postMessage({ ok: true, action: "opened", tabId: tab.id, url });
      return;
    }

    port.postMessage({ ok: false, error: "Unknown action" });
  } catch (e) {
    port.postMessage({ ok: false, error: String(e) });
  }
});

port.onDisconnect.addListener(() => {
  // se o host cair, Firefox desconecta
  // você pode tentar reconectar aqui se quiser
});
```

## 2) Carregar a extensão no Firefox (temporário)
- Abra `about:debugging#/runtime/this-firefox`
- “Load Temporary Add-on…” e selecione o `manifest.json` dessa pasta.  
(Depois, se quiser permanente, você empacota/sideload, mas para MVP temporário basta.)

# Parte B — Native Host em Python (Windows 11)

## 3) Script do host: `native_host.py`
Este host fala com o Firefox via stdin/stdout usando o framing padrão do Native Messaging (4 bytes little-endian + JSON). Isso é o padrão documentado para Native Messaging. [developer.mozilla](https://developer.mozilla.org/en-US/docs/Mozilla/Add-ons/WebExtensions/Native_messaging)

```python
import sys
import json
import struct
import threading
import socket

HOST = "127.0.0.1"
PORT = 8765  # porta local para o seu robô chamar

# ---- Native Messaging framing ----
def read_native_message():
    raw_len = sys.stdin.buffer.read(4)
    if len(raw_len) == 0:
        return None
    msg_len = struct.unpack("<I", raw_len)[0]
    data = sys.stdin.buffer.read(msg_len)
    if not data:
        return None
    return json.loads(data.decode("utf-8"))

def send_native_message(obj):
    data = json.dumps(obj).encode("utf-8")
    sys.stdout.buffer.write(struct.pack("<I", len(data)))
    sys.stdout.buffer.write(data)
    sys.stdout.buffer.flush()

# ---- TCP server for your automation ----
def handle_client(conn):
    try:
        buf = b""
        while True:
            chunk = conn.recv(4096)
            if not chunk:
                return
            buf += chunk
            # protocolo simples: 1 linha JSON por request, terminada por \n
            while b"\n" in buf:
                line, buf = buf.split(b"\n", 1)
                line = line.strip()
                if not line:
                    continue
                req = json.loads(line.decode("utf-8"))

                # Repassa para a extensão
                if req.get("action") == "open_url":
                    send_native_message({"action": "open_url", "url": req.get("url", "")})
                    # Resposta imediata: "sent". A confirmação real virá do Firefox e será repassada via outro canal se você quiser.
                    conn.sendall((json.dumps({"ok": True, "status": "sent"}) + "\n").encode("utf-8"))
                else:
                    conn.sendall((json.dumps({"ok": False, "error": "unknown action"}) + "\n").encode("utf-8"))
    except Exception as e:
        try:
            conn.sendall((json.dumps({"ok": False, "error": str(e)}) + "\n").encode("utf-8"))
        except:
            pass
    finally:
        conn.close()

def tcp_server():
    s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
    s.bind((HOST, PORT))
    s.listen(5)
    while True:
        conn, _ = s.accept()
        threading.Thread(target=handle_client, args=(conn,), daemon=True).start()

def main():
    # inicia servidor TCP para seu robô
    threading.Thread(target=tcp_server, daemon=True).start()

    # loop lendo mensagens da extensão (ACK/erros)
    while True:
        msg = read_native_message()
        if msg is None:
            break
        # por enquanto, só loga; depois a gente repassa ACK via TCP se quiser
        # (ex.: manter lista de requests por correlation_id)
        # importante: não use print() aqui (stdout é o canal do native messaging).
        # se precisar logar, use stderr:
        sys.stderr.write(f"[native-host] from extension: {msg}\n")
        sys.stderr.flush()

if __name__ == "__main__":
    main()
```

### Como seu robô chama (exemplo)
O seu Selenium/Python principal manda uma linha JSON para `127.0.0.1:8765`:
```python
import socket, json

def open_infojud_url(url):
    s = socket.create_connection(("127.0.0.1", 8765), timeout=5)
    s.sendall((json.dumps({"action":"open_url", "url": url}) + "\n").encode("utf-8"))
    resp = s.recv(4096).decode("utf-8").strip()
    s.close()
    return resp
```

# Parte C — Registrar o Native Host no Windows (Firefox)

## 4) Crie o manifesto do host (JSON)
Arquivo:  
`C:\Users\<SEU_USUARIO>\AppData\Roaming\Mozilla\NativeMessagingHosts\br.com.seuapp.infojud_bridge.json`

Conteúdo:
```json
{
  "name": "br.com.seuapp.infojud_bridge",
  "description": "InfoJud Bridge Native Host",
  "path": "C:\\caminho\\completo\\para\\python.exe",
  "type": "stdio",
  "allowed_extensions": ["infojud_bridge@local"]
}
```

### Observações importantes
- `path` deve apontar para o **python.exe** que vai rodar o `native_host.py` (ex.: do seu venv) **OU** você pode apontar para um `.exe` empacotado (pyinstaller) para simplificar.
- No Firefox, `allowed_extensions` precisa ser o **ID da extensão**. Como em MV3 temporário isso pode variar, o jeito mais fácil é **definir um ID fixo**.

## 5) Definir ID fixo na extensão (recomendado)
Edite o `manifest.json` e adicione:
```json
"browser_specific_settings": {
  "gecko": {
    "id": "infojud_bridge@local"
  }
}
```

Ficaria (mesclando com o manifest acima):
```json
{
  "manifest_version": 3,
  "name": "InfoJud Bridge",
  "version": "0.1.0",
  "description": "Recebe URLs via Native Messaging e abre no Firefox normal.",
  "permissions": ["nativeMessaging", "tabs"],
  "host_permissions": ["<all_urls>"],
  "background": { "scripts": ["background.js"] },
  "browser_specific_settings": {
    "gecko": { "id": "infojud_bridge@local" }
  }
}
```

Agora o `allowed_extensions` do host pode ficar fixo como `"infojud_bridge@local"`.

# Parte D — Rodar tudo

## 6) Ordem de execução
1) Carregue a extensão no Firefox (about:debugging).  
2) Inicie o native host:
   - Opção simples: executar manualmente:
     - `python native_host.py`
   - (Depois você pode rodar como serviço/daemon do Windows)
3) Do seu robô Selenium, chame `open_infojud_url(url)`.

Se tudo estiver ok:
- O Firefox abrirá uma nova aba com a URL enviada.
- O native host vai receber o ACK `{ok:true, action:"opened"...}` no stderr (por enquanto só log).
